# Governance

Bookmark Maxxing is currently a maintainer-led open-source framework.

## Maintainer

- Ben Newell (`@bennewell35`)

## Project Direction

The project is framework-first.

The current priority is:

1. Clear method.
2. Useful prompts.
3. Reusable templates.
4. Strong examples.
5. Source attribution.
6. Lightweight tooling only when the workflow is proven.

This is intentionally not an app yet.

## Decision Rules

Contributions are more likely to be accepted when they:

- make the framework easier to use
- improve source attribution
- improve contributor safety
- turn repeated patterns into reusable skills
- add examples that others can copy
- stay read-only with respect to source platforms

Contributions are less likely to be accepted when they:

- add broad product scope too early
- require credentials, cookies, or private scraping
- turn the repo into generic AI-news aggregation
- add vendor-specific automation without a clear proof gate
- duplicate existing prompts or templates

## Maintainer Authority

The maintainer has final decision authority on scope, releases, contribution acceptance, and moderation.

This may change later if the project gains regular contributors.

## Release Model

For now, changes land through normal pull requests into `main`.

No semantic versioning is promised yet. Tags can be added once the framework stabilizes.
