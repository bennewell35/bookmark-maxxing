# Bookmark Maxxing Framework

Bookmark Maxxing converts saved links into reusable skills.

The framework has six steps:

```text
Collect -> Extract -> Cluster -> Convert -> Publish -> Compound
```

## 1. Collect

Choose a bounded source window.

Examples:

- X bookmarks from the last week
- last 100 GitHub stars
- saved Slack posts from one channel
- newsletter links from one month
- browser bookmarks about one topic

Collection should be read-only.

Do not like, repost, reply, follow, subscribe, unbookmark, or mutate anything while collecting.

## 2. Extract

Capture structured source data:

- source
- author
- URL
- date
- title or post text
- linked artifact
- why it matters

The point is not to preserve everything. The point is to preserve enough to reason about patterns.

## 3. Cluster

Group items by repeated ideas.

Useful clusters usually show up across multiple people, sources, tools, or examples.

Weak clusters are usually:

- one-off hype
- generic advice
- motivational content
- vendor announcements with no reusable behavior

## 4. Convert

Every useful theme should become a reusable behavior.

Conversion examples:

```text
Theme: Agents need better memory.
Skill: Convert repeated lessons into docs, tests, skills, or runbooks.
```

```text
Theme: AI coding is shifting toward loops.
Skill: Design repeatable systems, not better prompts.
```

```text
Theme: Saved context is scattered.
Skill: Make your saved context readable by the systems doing the work.
```

If the theme does not change future behavior, it is not a skill yet.

## 5. Publish

The public artifact should lead with a concrete transformation.

Examples:

- I turned 105 bookmarks into 5 reusable agent skills.
- I turned my saved posts into a local knowledge workflow.
- I converted a week of AI engineering links into operating principles.

Avoid generic AI-news framing.

Use:

```text
What My Bookmarks Say About [Topic] This Week
```

instead of:

```text
What Happened in AI This Week
```

## 6. Compound

The article is not the final asset.

After publishing, convert the issue into:

- skill cards
- prompts
- checklists
- scripts
- repo issues
- product ideas

That is how bookmarks become infrastructure.

## Quality Bar

A Bookmark Maxxing issue is good if:

- the source window is clear
- sources are attributed
- ignored categories are explicit
- every theme becomes a skill
- the reader can reuse at least one workflow immediately
- the ending reinforces the frame: bookmarks are the input, skills are the output
