# Code of Conduct

Bookmark Maxxing is intended to be useful, practical, and respectful.

This project is about turning saved links into reusable skills. The community should reflect that: clear attribution, low ego, and concrete improvement.

## Expected Behavior

- Be respectful and constructive.
- Attribute sources accurately.
- Critique ideas and artifacts, not people.
- Keep discussion relevant to the framework.
- Assume good intent, but accept correction quickly.
- Help contributors turn broad ideas into usable workflows.

## Unacceptable Behavior

- Harassment, threats, personal attacks, or discriminatory language.
- Publishing private, sensitive, or doxxing information.
- Misrepresenting another person's work or endorsement.
- Spam, engagement farming, or low-effort promotion.
- Pressuring maintainers to accept unrelated features or app rewrites.
- Using the project to scrape, expose, or mutate private saved content.

## Enforcement

Maintainers may remove comments, close issues, block users, or reject contributions that violate this code.

For now, enforcement is handled by the project maintainer, Ben Newell.

## Reporting

If you need to report a conduct issue, contact the maintainer privately through GitHub.

Do not open public issues for private conduct reports.

## Attribution

This code of conduct is inspired by common open-source community-health practices, including the Contributor Covenant, but is written specifically for this project.
